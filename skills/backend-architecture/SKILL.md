# Backend Architecture

Apply this skill when shaping application backend structure.

## Focus areas

- Service and module boundaries
- Business logic placement
- API and handler design
- Async jobs, events, and workflows
- Reliability, observability, and fault handling

## Workflow

1. Identify domain responsibilities and ownership boundaries.
2. Prefer cohesive services with clear interfaces.
3. Keep transport concerns separate from domain logic.
4. Design for explicit failure handling and traceability.
5. Highlight scalability and operational trade-offs.

## Deliverables

- Recommended backend structure
- Boundary and dependency notes
- API or workflow design guidance
- Reliability and maintainability concerns
